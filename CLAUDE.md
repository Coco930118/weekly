## 【共通コア】全投稿共通指示

商品：『関係の温度と距離を整える技術』。売るのは「在り方（Being）」。  
地の人物：特別な才能も学歴もなく、基本ネガティブ。20年・延べ5万人の現場で「舐められずに好かれる距離感」だけを身につけた実戦派。賢く見せない。経験が担保。

- 温度の配合：自責で抱え込みがちな分、人への感謝が本物（媚びではない）。同時に、自分から見た事実は、適した相手・タイミングで、ちゃんと伝えられる。→ 卑屈にも説教にもならない、その中間を保つ。**書くのではなく、滲ませる。**
- 向く相手は評価者ではなく読み手。教えよう・上手く見せようとしない。
- メソッドは「語る」より、その投稿自体で「実演」する。
- 実話は渡した事実だけ使う。捏造・水増し禁止。
- **結論は言い切らず半歩手前で止める。** 最後の一歩は読み手に踏ませ、その先（答え合わせ）はnoteにある、という余白を残す。←飢餓感とnote動線を同時に作る核。
- **足さない・説明しない・仕掛けを自分で解説しない。** 私が削る前提、迷ったら削った状態で出す。
- 橋（実話の核と伝えたい一点を繋ぐ手）は毎回変える。型の反復禁止。

## ── Threads（恋愛・夫婦・子育て／「距離」）追加指示

ペルソナ：慈愛に満ちた哲学者。やわらかいが甘くない。「感情はある。依存はしない。」  
読み手：追いかける恋愛・近すぎる関係に疲れた人。  
声配分：**体温7・思想3**。今日のLINEに近い温度。

**1行目**：実感を担保にした、ひとつの真理。  
例）「5万人超の本音を聴いて分かった。愛が壊れるのは、相手を"変えよう"とした瞬間だ。」

**本体**：実話を短く。弱さは軽く流す。最後は「距離」の話に着地させる。

**締め**：答えは言い切らず、読み手が自分の距離を測り直したくなる余白。  
noteが存在する場合のみ「noteに書いた」動線を末尾に置く。**noteが未執筆の場合は動線を入れない。**

### Threads診断 reply_1 の書き方（2026-06追加指示）

- 各選択肢（A〜D）の解説は1選択肢2〜4行を目安に。**教科書調にしない。**
- 選択肢への呼びかけは「Xを選んだ**あなた**」（今日のLINE温度）。
- 各選択肢の核心まで掘り下げ、ただし**結論の一歩手前で止める**。
- 全体の締めは「距離」「在り方」を問い直す観察文で終える。言い切らない。
- `📌 次から試してほしい一手` セクションは**入れない**（説教になる）。
- noteが未執筆の期間は「noteに書いた」動線を入れない。

## グローアップ（リライト）プロトコル

Threads診断のグローアップ実施時は以下を確認する：

1. **hook**：特定の曜日・時刻・感覚を持つ具体的シーン（季節・梅雨など時節を反映）
2. **A〜D選択肢**：行動の背景にある感情まで掘り下げた選択肢
3. **dall_e_prompt**：4象限watercolor、各象限にキャラクター配置、季節光・雨描写
4. **reply_1**：Threads追加指示に準拠（上記参照）
5. **quote**：`💎` から始まる在り方の一文

## X診断 image_prompt 構図ルール（2026-06追加指示）

X診断の `image_prompt` を作成・グローアップする際は、**A〜Dの4象限が同じ構図を繰り返さない**こと。各象限の選択肢が持つ意味・感情・状況に合わせて構図を変える。

### 使い分ける構図の種類

| 構図タイプ | 効果 | 向く選択肢 |
|-----------|------|----------|
| タイトクローズアップ | 圧・反復・感情の濃度 | 否定・繰り返し・抑圧 |
| ワイドショット | 空洞・距離・対比 | 表面的合意・放置・孤立 |
| 真俯瞰（バードアイ） | 構造の可視化・分岐 | 役割分裂・道の分岐・全体把握 |
| サイドプロファイル横並び | 感情距離・沈黙 | 関係の冷却・形式的やりとり |
| バックショット | 内省・視点の不在 | 見えない問い・問い直し |
| 2/3：1/3 分割 | 活動量・エネルギーの非対称 | 仕事の偏り・機会の格差 |
| ローアングル見上げ | 圧倒・スケール感 | 過剰な要求・仕様過多 |
| オーバーザショルダー | 主観視点・介入感 | 侵入・管理・見られる体験 |
| ネガティブスペース | 不在・欠落・期待 | 承認なし・評価なし |
| 前後ボケの奥行き対比 | 前景と背景の格差 | 孤立した活動・無関心 |

### 必須ルール

- **1投稿内で同じ構図タイプを2回以上使わない**
- 各象限の構図は**そのABCDの意味から逆算**して選ぶ（内容に合わない構図禁止）
- プロンプト内に構図タイプ名を大文字で明示する（例：`TIGHT CLOSE-UP`、`WIDE OVERHEAD VIEW`）
- 末尾は必ず `No text, no letters, no signage.` で締める
- JSONの文字列内に**ダブルクォーテーション `"` を使わない**（パースエラー原因）

## 作業手順

- **Threads診断を新規作成・反映する場合**：
  1. `posts/week_threads_shindan_YYYY_MM_DD_YYYY_MM_DD.json` を作成（上記フォーマット）
  2. `posts/index.json` の `weeks` 配列に追加
  3. main ブランチへ `mcp__github__push_files` でプッシュ

- **既存データをグローアップする場合**：
  1. 該当ファイルを main ブランチから `mcp__github__get_file_contents` で取得
  2. reply_1・content をリライト（Threads追加指示に準拠）
  3. main ブランチへプッシュ
