# coconocanvas 週次投稿管理サイト — 運用メモ

## 毎週の実行手順

### タイミング
毎週土曜（または依頼時）に次週分（火〜月）を生成・プッシュする。

### 手順

1. **前々週のデータを削除**
   - `posts/week_YYYY_MM_DD_YYYY_MM_DD.json`（2週前のファイル）を削除
   - `posts/index.json` から該当エントリを削除

2. **新週のJSONを1日ずつ生成・プッシュ**（計7回プッシュ）
   - ファイル名：`posts/week_YYYY_MM_DD_YYYY_MM_DD.json`
   - 1日5投稿（X×2 + Threads×3）
   - 各日のコミット後に `git push origin <branch>`

3. **index.json に新週を追加**（最初の日に一緒に実施）

4. **main へ PR 作成→マージ**

### 削除ルール例
| 作成する週 | 削除する週 |
|-----------|-----------|
| 5/5〜5/11 | 4/21〜4/27 |
| 5/12〜5/18 | 4/28〜5/4 |
| 5/19〜5/25 | 5/5〜5/11 |

### 投稿構成
- **X**：6:00 🐢しずく / 22:00 🕊ひより（各1日2投稿）
- **Threads**：7:00 🐢しずく / 10:00 🐈‍⬛しらたま / 19:00 🕊ひより（各1日3投稿）
- 合計：35投稿／週

## Threads 投稿ルール（必須・2026年版）

### 前提
Coco Methodologyの世界観を保ちながら、20代前半〜30代の層にも"自然に自分ごと化される"投稿にする。
「共感と余白」最優先。バズよりも"また読みたくなる空気"を重視する。

### 重要方針
内容のレベルは下げない。「入口の理解コスト」を下げる。
"わかる人に届ける"ではなく、"まだ言語化できていない人に気づかせる"設計にする。

### 禁止事項
- 内容を浅くする
- 説明しすぎる（解説過多）
- 正解を押しつける
- スピリチュアル・自己啓発っぽくする
- 強い断定・ジャッジ
- 「あなた」「みんな」など直接的な呼びかけ

### 構成ルール（必須）

**① 冒頭1行で「静かな共感 or 違和感」**
- 誰でも感じたことがある微細な感情から入る
- 問いかけ or つぶやき形式
- 強すぎない、でも引っかかる一文

**② 中盤で「ズレの気配」をにじませる**
- 説明しすぎない
- 少しずつ"違和感の正体"に触れる

**③ 後半で「構造」を一瞬だけ出す**
- 温度差／距離／認識のズレなど、Coco Methodologyの視点を1文以内で静かに触れる
- 断定せず「そんな気配がある」「ふと気づく」くらいの柔らかさ

**④ 最後は「余白」で終わる（最重要）**
- 答えを閉じない
- 読者が自分の中で考えられる状態にする

### 追加ルール（2026年アルゴリズム対応）
- 最後に「余白を残しつつ、自然に返信したくなる軽い問い」を1行以内で入れる（推奨）
- 一文は15〜20文字以内を強く意識
- 絵文字は世界観を崩さない最小限（温度感を補う程度）

### 質問設計ルール
強制感のない自然な問いにする。思い出や感覚を引き出す形にする。

OK例：「この感覚、最近感じたのはいつ」「どの瞬間に違和感が出た」
NG例：「どう思う？」「みんなはどう？」

### 質問パターンのローテーション（必須）
同じ問いの構造が連続しないよう以下をローテーション。同一パターンは2投稿以上連続禁止。直近7投稿内で2回以上使用禁止。

| パターン | 例 |
|---------|---|
| ① 時系列系 | 最近これを感じたのはいつだった |
| ② シーン想起系 | どの場面でそれが出た |
| ③ 比較・違和感系 | 以前と何か違う感じがあった |
| ④ 内面観察系 | 自分の中で何が動いていた |
| ⑤ 行動振り返り系 | そのとき何をしていた |
| ⑥ 関係性視点系 | 誰かとの間でそれを感じた |
| ⑦ 無意識気づき系 | 気づかないうちにそうしていた |

### 文章スタイル
- 最大10行以内
- 1行1〜2文（短く切る）
- 改行でリズムを作る
- 詩的すぎず、リアル寄り
- 感情は「冒頭の違和感」で最も強く出す

### 言葉のルール
- 抽象ワード（温度差・距離など）は後半に置く
- 最初は具体的な体験・感情
- 専門用語は使いすぎない
- 説明より"気づきの余韻"を優先

### 正しい流れ
感情 → 違和感 → 気配 → 構造 → 余白 → 軽い問い

### NGパターン
- いきなり概念から入る（例：関係の温度差は〜）
- 説明で完結する投稿
- 読み終わって何も残らない

### 目的
- 読者が「これ、自分のことかも」と静かに気づくこと
- 深く共感されること
- 何度も読み返されること
- 自然に返信・対話が生まれること
