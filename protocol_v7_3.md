## ■ STEP 0：グローアップ分析（毎週土曜12時実行）

前週35投稿を構造分析し、以下を抽出する。

### 負けパターン判定基準
- 禁止ワード「設計」の使用（quote欄含む全文チェック）
- XにThreads領域コンテンツの混入
- 締めパターンの固定化（二択が3件連続以上）
- Threadsのまとめすぎ・説明過多（最後2行で解説している場合は削る）
- X投稿の現場の泥臭さゼロ（構造指摘のみで後味の悪さがない）

### エラーフォールバック（必須）
① 実データ取得不可 → 構造のみで分析（数値不使用）  
② 分析途中停止 → 完了済み範囲から勝ち/負けを抽出、未分析は補完推定  
③ 投稿単位で評価不能 → 「フック・展開・締め」構造単位で分析  
④ 改善提案が出せない → 固定軸（解像度・体温・余白・構造/感情分離・現場の泥臭さ）から3つ選択  
⑤ 全体不可 → 前週プロトコルとの差分比較のみで出す

## ■ STEP 4：痛みの再定義プロトコル

### 3段階の掘り下げ構造
```
表層の感情（疲れた・傷ついた・しんどい）
  ↓ 再定義
その感情が生まれた構造・関係・記憶
  ↓ 再定義
実は何を守ろうとしていたか / 何を恐れていたか
```

### X版：構造の再定義
- 「〜が問題だ」ではなく「〜が起きているとき、その下には〜がある」
- 現場の違和感を「仕組みの不在」として言語化する
- 根性論を「仕組みへの逃避」として静かに解体する
- **必須：現場の泥臭さを一行入れる**（判断の迷い・後味の悪さ・誰かが払っているコスト）

### Threads版：感情の再定義
- 「〜が悲しい」ではなく「〜は実は〜だった」
- 感情の名前を使わず、身体感覚・状態・場面で描写する
- 「指先が冷える」「心臓がぎゅっとする」「重さが降りてくる」など生理的な描写を使う

## ■ STEP 5：映像の喚起プロトコル

### 解像度の必須チェック項目
- 時刻（深夜2時 / 水曜の夜 / 月曜の朝）は具体か
- 場所（廊下 / 会議室の後 / スマホの画面）は描かれているか
- 動作（打ちかけて消した / 下書きに保存した / 静かに降りてきた）は見えるか
- 生理的描写（指先が冷える / 心臓が重くなる / 肩が落ちる）はあるか

### 禁止：抽象表現のリスト
- 「大切な存在」→「深夜2時に名前を打ち込んだ相手」
- 「関係が壊れた」→「また今度、が積み重なった末の小さな諦め」
- 「コミュニケーション不足」→「廊下でやっと出てくる本音」
- 「自己成長」→ 使用禁止（文脈で具体的に描く）

## ■ STEP 6：体温の宿し方プロトコル

### X版：体温の出し方
- 断言＋余韻（「〜だ」で終わるが、その後に問いを置く）
- 一人称の経験を匂わせる（「現場でよく見る」「20年でわかった」）
- **現場の泥臭さ**：「その泥の中で誰かが何かを黙って払っている」感覚を一行入れる

### Threads版：体温の出し方
- 揺らぎのある語尾（「気がする」「かもしれない」「と思ってる」）
- 美しくない本音の一滴（整えられていない感情・矛盾・迷い）
- 語りかけない（語りかけると説明になる。独白で書く）
- **余白**：最後の1行は言い切らない。読者が続きを想像できる終わりにする

## ■ STEP 7：プラットフォーム別ルール（厳格分離）

### X投稿ルール

**コンテンツ領域（この領域のみ扱う）**
- 組織・チーム・職場の構造問題
- リーダーシップ・マネジメントの仕組み
- 根性論 vs 仕組みの対比
- 会議・評価・報告・指示・フィードバックの構造

**禁止コンテンツ（Threadsへ移動）**
- 人間関係の距離感・余白・境界線
- 恋愛・個人的な感情の機微
- 「送れなかった言葉」系の個人的独白
- 「眠れた」「夕飯」などの日常系

**フォーマット**
- 文字数：4〜6行（簡潔・刺さる）
- 締めパターン（以下を交互に使用。同一パターン3件連続禁止）
  - 二択型：「〜か」「〜か」▶
  - 余韻型：一文の事実で終わる（「その声は、廊下に消えた」）
  - 問いかけ型：単一の問い（「誰が気づいているか、問い直す価値がある」）
  - 事実提示型：問いなしで現象だけ置く
- 💎マークを1箇所使用
- quoteは「💎 〇〇、〇〇だ」形式（「設計」使用禁止）

**現場の泥臭さ必須チェック**
- 投稿ごとに「判断の迷い・違和感・後味の悪さ・誰かが払っているコスト」が一行でも乗っているか確認する
- 抽象的な構造指摘のみで終わっている場合は必ず修正する

### Threads投稿ルール

**コンテンツ領域（この領域のみ扱う）**
- 人間関係の距離感・余白・境界線
- 静かな疲れ・送れなかった言葉・消した気持ち
- 丁寧な大人女性のこぼれた言葉
- 曜日の心の温度と連動した内省

**禁止コンテンツ（Xへ移動）**
- 組織構造・職場のマネジメント論
- リーダーシップの仕組み論
- 会議・評価などの職場テーマ

**フォーマット**
- 文字数：8〜14行（余白を持たせる）
- 空行を使い、呼吸できる構造にする
- 締め：読者に解釈を委ねる余韻で終わる（解説・まとめ禁止）
- 💍マークを1箇所使用
- 末尾に統一署名（キャラクターによって異なる）
- quoteは「💍 〇〇、〇〇」形式（言い切らない余韻を持たせる）

**共犯感・余白チェック（必須）**
- 「わかる」で終わっていないか（共犯感：一緒に抱えている感覚があるか）
- 最後の1行を言い切っていないか（余白：読者が続きを想像できるか）
- 説明過多・まとめすぎ・綺麗すぎになっていないか（削れるものは削る）

**統一署名（しずく・しらたま・ひより共通）**
```
感情はある。依存はしない。そんな距離の整え方を、ここに残していく💍
```
※返信獲得目的の投稿は署名なしで終わることも可（余韻優先の場合）

```
/posts/
  index.json            ← 週一覧（最新2週分のみ）
  week_YYYY_MM_DD_YYYY_MM_DD.json  ← 週次データ
/protocol_v7_3.md       ← 本プロトコル
```

### データ管理ルール

- 各投稿JSONは `week_開始日_終了日.json` 形式

### JSON更新手順
1. 前々週ファイルを削除
2. 新週ファイルを作成（35投稿）
3. index.jsonを更新（2ファイル分のみ）
